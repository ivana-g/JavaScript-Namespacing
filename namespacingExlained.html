Namespacing Fundamentals

Single Global Variable Pattern

Opting for primary object of reference.
Make sure that no one else has used the same global variable.
Prefix Namespacing - unique prefix namespace (ex. "myApplication_") and then define any methods, variables or other objects after the prefix. (This again is relying on unique name for the namespace prefix and the name of the variables, methods.)
singleGlobalVariables.js

Object Literal Notation

Object containing a collection of key:values pairs, similar to a normal array.
Easily readable structures. Used for decoupling a default configuration for an application. This way the default configuration is located in a single area that can be easily modified without the need to search throught the entire codebase.
Have the potential to grow in long syntactic constructs.
objectLiteralNotation.js

Nested Namespacing

Extension of the object literal patern. Low risk of name collision. Relies on the low chance that nested children already exist even if the namespace already exists.
nestedNamespacing.js

Immediately-infoked Function Expressions (IIFE)

An unnamed function which is immediately invoked after it's beeing defined.
iife.js

Namespace injection

Variation of IIFE where the methods and properties are injected for a specific namespace from wihin a function wrapper using this as a namespace proxy.
Easy application of functional behavior to multiple objects or namespaces and can come in useful when applying a set of base methods to be built on later (eg. getters and setters).
There might be more optimal approached to achive the functional behavior ex. deep object extension / merging.
namespaceInjection.js
